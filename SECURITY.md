# Security Policy

## Reporting a vulnerability

请不要在公开 Issue 中披露未修复的安全问题。请通过本仓库 GitHub Security 页面的“Report a vulnerability”入口提交私密报告，并说明受影响版本、复现步骤、影响范围和建议缓解措施。

请勿在报告中提交真实配对令牌、Cookie、DNSPod 密钥、证书私钥、声纹向量或用户音频。收到报告后，维护者会先确认问题和影响，再协调修复与披露时间。

## Deployment responsibility

本项目是自托管软件。部署者需要自行保护公网入口、TLS 私钥、模型接口、日志、会话数据库和声纹档案，并确保只有预期用户可以访问服务。
